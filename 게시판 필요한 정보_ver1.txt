 아이디의 관리 ?
 mysql 에 사용가능한 auto_increment
 -> 자동으로 id 를 1씩 증가시켜서 넣어줌
 -> 1 2 3 에서 2가 사라져도 다음 id 는 자동으로 4로 맞춰줌
 -> 다른 데이터 테이블에 id 를 저장한다고 해도 중복되거나 꼬일 일은 없음
 근데 댓글과 같이 수가 늘어나는 속도가 빠른 애들은 id 값이 얼마 안지나서 엄청 커질꺼임
 신경안쓰고 그냥 진행? or id 를 가리키는 필드를 따로 만들어서 지정을 해주는 방법이 필요함
 단, 댓글을 게시글모델인 Info 안에 넣는다면 댓글에 한해서는 id 그냥 auto_increment 해도 가능할듯
 
 근데 게시글모델안에 댓글을 넣어버릴 때 큰 단점이 있음
 프로필상에서 내가 작성한 댓글들 조회하기가 매우 빡셈

1) 게시글 안에 댓글들 저장
    작성한 댓글 조회 -> 모든 게시글들을 돌면서 내 닉네임과 매치하는 댓글들 가져옴
    물론 얘는 프로필에 딕셔너리 형식으로 { 게시글 아이디 : 댓글 아이디 } 처럼 저장하면되긴 함
2) 게시글 밖에 댓글들만 따로 저장
    작성한 댓글 조회 -> 그냥 아이디에 맞는 댓글 긁어오면 그만
    얘로 하면 그냥 프로필에 리스트로 댓글 아이디들만 넣으면됨
